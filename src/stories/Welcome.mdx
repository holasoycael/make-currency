## 🧩 Exports Overview

####

<Coding.ExportsOverview />

- **`currency()`**
  Formats a numeric value into a locale-aware currency string. Supports custom symbol, empty-state, and currency config.

- **`currencyFn()`**
  Parses a formatted string and returns both `floatValue` and `stringValue`. Ideal for real-time input masks.

- **`CONFIGURE()`**
  Sets the global default currency. All subsequent calls to `currency()` and `currencyFn()` will use it.

- **`TYPES`**
  Object containing all 139 currency configurations, accessed by ISO 4217 code: `TYPES.USD`, `TYPES.BRL`, etc.

## 🔄 How It Works

<div
  style={{

>
  <code>Input Value (1234.56)</code>
  <span style={{ margin: 0 }}>↓</span>
  <code>Intl.NumberFormat (locale + currency)</code>
  <span style={{ margin: 0 }}>↓</span>
  <code>replaceSymbol (optional transform)</code>
  <span style={{ margin: 0 }}>↓</span>

> **💡 Tip:** The library uses `Intl.NumberFormat` under the hood, which means formatting automatically adapts to the correct decimal separator, thousands grouping, and symbol placement for each locale.
